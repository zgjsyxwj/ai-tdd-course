# 用户管理系统 需求规格说明

> 版本：v1.0 | 日期：2026-04-12 | 状态：已批准

## 1. 概述

### 1.1 业务背景

系统需要一个基础的用户管理模块，支持用户的注册、查询、密码修改和删除。

### 1.2 目标

提供用户 CRUD 的 REST API，包含基本的参数校验和唯一性约束。

### 1.3 范围

- **包含**：用户注册、单个查询、列表查询、修改密码、删除
- **不包含**：登录认证、权限管理、分页、排序

---

## 2. 实体模型

### 2.1 User

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | int | PK, 自增 | 主键 |
| username | varchar(10) | 必填, 唯一 | 用户名 |
| pwd | varchar(30) | 必填, 最小6位 | 密码 |

---

## 3. API 规格

### 3.1 POST /api/users — 用户注册

**Request Body:**

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|----------|------|
| username | string | 是 | 长度 1-10 | 用户名 |
| pwd | string | 是 | 最小长度 6 | 密码 |

**Success Response:** `201 Created`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 用户 ID |
| username | string | 用户名 |

> 注意：响应中**不返回** pwd 字段。

**Error Responses:**

| 状态码 | 触发条件 | 说明 |
|--------|----------|------|
| 400 | username 为空 | 参数校验失败 |
| 400 | pwd 为空 | 参数校验失败 |
| 400 | pwd 少于 6 位 | 参数校验失败 |
| 409 | username 已存在 | 用户名冲突 |

---

### 3.2 GET /api/users/{id} — 查询用户

**Path Parameters:**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | int | 用户 ID |

**Success Response:** `200 OK`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 用户 ID |
| username | string | 用户名 |

> 不返回 pwd。

**Error Responses:**

| 状态码 | 触发条件 | 说明 |
|--------|----------|------|
| 404 | 用户不存在 | |

---

### 3.3 GET /api/users — 用户列表

**Success Response:** `200 OK`

返回 JSON 数组，每个元素：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 用户 ID |
| username | string | 用户名 |

> 不返回 pwd。无分页。

---

### 3.4 PUT /api/users/{id} — 修改密码

**Path Parameters:**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | int | 用户 ID |

**Request Body:**

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|----------|------|
| pwd | string | 是 | 最小长度 6 | 新密码 |

> 只能修改密码，不能修改 username。

**Success Response:** `200 OK`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 用户 ID |
| username | string | 用户名 |

**Error Responses:**

| 状态码 | 触发条件 | 说明 |
|--------|----------|------|
| 400 | pwd 少于 6 位 | 参数校验失败 |
| 404 | 用户不存在 | |

---

### 3.5 DELETE /api/users/{id} — 删除用户

**Path Parameters:**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | int | 用户 ID |

**Success Response:** `204 No Content`

无响应体。

**Error Responses:**

| 状态码 | 触发条件 | 说明 |
|--------|----------|------|
| 404 | 用户不存在 | |

---

## 4. 业务规则

| 编号 | 规则 | 说明 |
|------|------|------|
| BR1 | username 全局唯一 | 通过数据库 UNIQUE 约束 + Service 层检查双重保障 |
| BR2 | 响应不返回 pwd | 所有 API 的响应中都不包含密码字段 |
| BR3 | 删除为硬删除 | 直接从数据库移除记录 |
